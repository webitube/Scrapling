## Environment Setup

### Prerequisites

- Python 3.10–3.13
- pip (latest)
- Git

### Initial Setup

```bash
# Clone and enter the repository
git clone https://github.com/D4Vinci/Scrapling.git
cd Scrapling

# Install the project with all extras in development mode
pip install -e ".[all]"

# Install test dependencies
pip install -r tests/requirements.txt

# Install browser dependencies (Playwright + Patchright)
scrapling install
```

## Code Quality Pipeline

Triggered on push/PR to `main` or `dev` (excluding markdown-only changes).

### Tools & Steps

| Step | Tool | Purpose |
|------|------|---------|
| 1 | **Bandit** | Security linting (`.bandit.yml` config) |
| 2 | **Ruff (linter)** | Style and correctness checks (`ruff.toml` config) |
| 3 | **Ruff (formatter)** | Formatting validation |
| 4 | **Vermin** | Python 3.10+ compatibility check |
| 5 | **Mypy** | Static type checking |
| 6 | **PyRight** | Static type checking (alternative) |

### Run Locally

```bash
# Bandit
bandit -r -c .bandit.yml scrapling/

# Ruff lint
ruff check scrapling/

# Ruff format check
ruff format --check scrapling/

# Vermin
vermin -t=3.10- --violations --eval-annotations --no-tips scrapling/

# Mypy
mypy scrapling/

# PyRight
pyright scrapling/
```

### Configuration Files

- `ruff.toml` — Ruff linter and formatter settings
- `.bandit.yml` — Bandit security skips
- `.pre-commit-config.yaml` — Pre-commit hook definitions

## Testing Pipeline

Triggered on push/PR to `main` or `dev`. Runs on macOS across Python 3.10–3.13 via Tox.

### Tox Configuration

Tox runs three sequential test phases to avoid conflicts:

1. **Browser tests** — `DynamicFetcher` and `StealthyFetcher` tests (no parallelization)
2. **Async tests** — Tests marked with `asyncio` (no parallelization)
3. **Remaining tests** — All other tests (parallelized with `pytest-xdist`)

### Run Locally

```bash
# Install tox
pip install tox

# Run all environments (all Python versions + pre-commit)
tox

# Run a specific Python version
tox -e py312

# Run only pre-commit checks
tox -e pre-commit
```

### Run pytest Directly

```bash
# All tests
pytest --cov=scrapling --cov-report=xml -v

# With parallelization
pytest -n auto -v

# Specific test file
pytest tests/parser/test_selector.py -v
```

### Configuration Files

- `tox.ini` — Tox environments and test commands
- `pytest.ini` — pytest settings (strict asyncio mode, verbose output)
- `tests/requirements.txt` — Test dependencies

## Docker Build & Push

Triggered when a PR is merged to `main` or manually via `workflow_dispatch`.

### Image Registries

| Registry | Image Name |
|----------|-----------|
| Docker Hub | `pyd4vinci/scrapling` |
| GitHub Container Registry | `ghcr.io/d4vinci/scrapling` |

### Build Details

- **Base image:** `python:3.12-slim-trixie`
- **Package manager:** `uv` (faster than pip)
- **Platforms:** `linux/amd64`, `linux/arm64`
- **Exposed port:** `8000` (MCP server HTTP transport)
- **Entrypoint:** `uv run scrapling`

### Tag Strategy

Automatic tags generated by `docker/metadata-action`:
- Branch name
- PR number
- Semantic version (`v1.2.3`, `1.2`, `1`)
- `latest` (on default branch)

### Required Secrets

| Secret | Purpose |
|--------|---------|
| `DOCKER_USERNAME` | Docker Hub login |
| `DOCKER_PASSWORD` | Docker Hub login |
| `CONTAINER_TOKEN` | GitHub Container Registry login |

### Build Locally

```bash
# Build the image
docker build -t scrapling:local .

# Run the container
docker run --rm scrapling:local --help

# Run with MCP server on port 8000
docker run --rm -p 8000:8000 scrapling:local mcp-server
```

### Dockerfile Optimization

The Dockerfile uses layer caching to speed up rebuilds:
1. `pyproject.toml` is copied first so dependency installation is cached
2. `uv` cache is mounted for faster installs
3. APT cache is mounted for system dependencies
4. Static version in `pyproject.toml` (not dynamic) enables better layer caching

## Release & PyPI Publish

Triggered automatically when a PR is merged to `main`.

### Release Workflow

```
PR merged to main
    │
    ├─→ Extract version from PR title (must start with 'v', e.g., 'v0.4.9')
    │
    ├─→ Create GitHub Release
    │     └─→ PR body becomes release notes
    │
    └─→ Build and publish to PyPI
          ├─→ python -m build --sdist --wheel
          └─→ pypa/gh-action-pypi-publish (trusted publishing)
```

### Steps to Release

1. Develop changes on a feature branch
2. Open a PR with the title in version format: `v0.X.Y`
3. Write release notes in the PR body
4. Merge the PR — the workflow handles the rest

### Requirements

- PR title **must** start with `v` (e.g., `v0.4.9`, `v1.0.0`)
- PyPI environment uses trusted publishing (`id-token: write`)
- No manual PyPI credentials needed

## Pre-commit Hooks

Three hooks run on every commit:

| Hook | Tool | Purpose |
|------|------|---------|
| 1 | **Bandit** | Security vulnerability scanning |
| 2 | **Ruff** | Linting with auto-fix |
| 3 | **Ruff Format** | Code formatting |
| 4 | **Vermin** | Python version compatibility |

### Setup & Run

```bash
# Install pre-commit
pip install pre-commit

# Install git hooks
pre-commit install

# Run manually on all files
pre-commit run --all-files

# Run on staged files only
pre-commit run
```

## Troubleshooting

### Common Issues

**Playwright browsers not found**
```bash
playwright install chromium
playwright install-deps chromium
```

**Tests fail with import errors**
```bash
pip install -e ".[all]"
pip install -r tests/requirements.txt
```

**Docker build fails on ARM**
Ensure Buildx is set up with QEMU for multi-arch:
```bash
docker buildx create --use --name multiarch
docker buildx inspect --bootstrap
```

**PyRight / Mypy report false positives**
Ensure `lxml-stubs` is installed:
```bash
pip install lxml-stubs
```

**Pre-commit hooks fail**
```bash
pre-commit clean
pre-commit install
```

### CI/CD Concurrency

Both the tests and code quality workflows use concurrency groups to cancel in-progress runs when new commits are pushed to the same branch:

```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
```

This prevents redundant CI runs and saves resources.
